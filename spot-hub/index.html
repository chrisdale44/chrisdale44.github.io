Crust
Central.

A Progressive Web App for discovering and mapping skateboarding spots worldwide — built with Next.js 15, Supabase, PostGIS, Deck.gl, and Gemini AI.

About

Crust Central is a PWA designed to map skateboarding spots and user-generated media on a global scale. Users can discover spots worldwide, log tricks performed at each location, and contribute their own discoveries.

The codebase is private for security reasons, but code samples are available on GitHub and full access can be granted on request.

Visit crustcentral.com ↗

Design Goals.

Performance

Optimisations.

WebGL via Deck.gl

Deck.gl is a WebGL-powered library that leverages GPU acceleration for significantly faster rendering and smoother interactivity, enabling Crust Central to handle thousands of map pins worldwide while maintaining smooth movement and scrolling.

Image Pipeline

  • CDN Delivery: Cloudinary provides low-latency image delivery worldwide.
  • Auto-optimisation: Images are compressed, converted to WebP, resized to 1000px maximum, and watermarked on upload.
  • Thumbnails: 200×200px thumbnails are auto-generated for fast-loading map pin previews.
  • Lazy loading: IntersectionObserver API + Next.js Image component reduce initial page load.
  • SVG icons: Lightweight, scalable, no HTTP requests.
  • Service Worker: Caches images in the browser cache API to reduce repeat requests.

Grid-Based Data Fetching

Crust Central uses a grid-based system for data fetching, where each grid square has a unique ID based on coordinates. On initial load, two concurrent requests are made:

  • Fetch shallow spot data worldwide — enough for map pins and thumbnails. Paginated if >5,000 spots (~1MB). Browsing and filtering is near-instantaneous once complete.
  • Pre-fetch complete spot data for grid squares surrounding the visible area. Grid IDs, timestamps, and cache depth are stored in IndexedDB to prevent duplicate requests.

As the user moves around the map, new grid squares are cross-referenced with shallow data to determine which ones contain spots before making database requests.

Local-First Caching Strategy

The flow for fetching spot data:

  • Check IndexedDB → if found and not stale, return data
  • Call Server Action (running on Netlify Edge)
  • Check Redis cache → if found and not stale, return data
  • HTTP fetch to Supabase → edge server checks in-memory cache first
  • Request made to Supabase as final fallback

User Experience

Features.

Image Geolocation

EXIF geolocation data is extracted from uploaded images to automatically position map pins at the correct location, streamlining the spot-adding experience.

Single spot upload with auto-geolocation

Batch Upload

Up to 50 images can be selected at once. Geolocation data automatically groups nearby images together. A drag-and-drop interface allows corrections before creating spots individually.

Batch upload with automatic grouping

AI Image Analysis (Gemini)

Google Gemini AI analyses spot images with a predefined prompt to generate suggested tags, enhancing the tagging process and improving discoverability.

Roadmap

Next Steps.

Beta Release Goals

  • Instagram API: Display related posts for each map pin.
  • YouTube API: Display related videos with timestamped playback.
  • Facebook Auth: Default login option (affects Instagram API limits).
  • PostHog: Web analytics, session recording, and A/B testing.
  • Stripe: Tiered pricing model for advanced features.
  • Vitest / RTL: Improve test coverage across codebase.
  • Playwright / Cypress: End-to-end test coverage.

Future Enhancements

  • Legend State or Zero Sync for local-first database syncing.
  • TypeScript for type safety as the app grows in complexity.
  • Remix or Waku as a more performant Next.js alternative.
  • Svelte, SolidJS, or Astro as a React alternative.
